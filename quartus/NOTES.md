# Quartus Synthesis Notes

Notes from the first successful local Quartus A&S run on this codebase.
Tested with **Quartus Prime Lite 17.0** targeting **Cyclone V (5CSEMA5F31C6)**.

---

## Tool and flow

**No sv2v needed.** Unlike the Icarus/cocotb simulation flow (which requires
sv2v to convert SystemVerilog down to Verilog-2005), Quartus 17 parses the
`.sv` source files natively. The synthesis flow reads `src/` and `synth/`
directly.

The relevant Makefile targets are:

```
make quartus_map    [QUARTUS_BIN=/path/to/bin64]   # Analysis & Synthesis only (~45 s)
make quartus_compile [QUARTUS_BIN=/path/to/bin64]  # Full flow: A&S → fit → asm → timing
```

`QUARTUS_BIN` defaults to empty (uses PATH). Override it if `quartus_map` is
not on your PATH:

```
make quartus_map QUARTUS_BIN=/path/to/intelFPGA_lite/17.0/quartus/bin64
```

Alternatively, invoke `quartus_map` directly from the `quartus/` directory
(which is where the project `.qsf` lives):

```
cd quartus
quartus_map de1_soc          # reads de1_soc.qsf automatically
```

## Source file set

The QSF (`quartus/de1_soc.qsf`) lists all files explicitly.  The set mirrors
the `compile_synth_top` Makefile target, minus the simulation-only wrappers:

**Include:**
- `src/alu.sv`, `src/controller.sv`, `src/dcr.sv`, `src/decoder.sv`
- `src/divergence.sv`, `src/dispatch.sv`, `src/fetcher.sv`, `src/lsu.sv`
- `src/pc.sv`, `src/registers.sv`, `src/scheduler.sv`, `src/core.sv`, `src/gpu.sv`
- `synth/mem_bridge.sv`, `synth/seg7.sv`, `synth/clock_step.sv`
- `synth/kernel_memories.sv` ← **must exist before running quartus_map**
- `synth/de1_soc.sv` (top-level)

**Exclude** (simulation-only, not synthesizable as-is):
- `synth/sim_program_rom.sv`
- `synth/sim_data_ram.sv`
- `synth/sim_harness.sv`

`kernel_memories.sv` is auto-generated by `make synth_kernel KERNEL=<test>`.
Any kernel image works for an elaboration check; the file checked into the
repo (generated from `test_diverge_ifelse`) is sufficient.

## Project files

`quartus/de1_soc.qsf` — hand-authored; contains FAMILY, DEVICE,
TOP_LEVEL_ENTITY, and the SYSTEMVERILOG_FILE list.  Paths are relative
(`../src/`, `../synth/`) so the project is portable across machines.

`quartus/de1_soc.qpf` — auto-generated by Quartus on the first `quartus_map`
run; small and safe to commit; required to open the project in the Quartus GUI.

`quartus/DE1-SoC.qsf` — pre-existing pin assignment file (LabsLand virtual
peripheral mapping).  Not needed for `quartus_map` (A&S), only for
`quartus_fit` (place-and-route).

```

  Peak virtual memory: ~5 GB
  Elapsed time: ~45 s
```

**Resource estimates (Cyclone V):**
| Resource       | Count |
|---------------|-------|
| Logic cells    | 11,224 |
| RAM segments   | 14    |
| DSP elements   | 8     |
| Input pins     | 15    |
| Output pins    | 52    |

**Inferences of note:**
- `program_rom` → `altsyncram` ROM (M10K block RAM, 256×14-bit).  The
  `initial` block in `kernel_memories.sv` is converted to a `.mif` file by
  Quartus automatically; no external `.mif` needs to be provided.
- `alu.sv` integer division (`/`, `%`) → 8 `lpm_divide` megafunctions
  (one per thread per core, with NUM_CORES=1 and THREADS_PER_BLOCK=4 that is
  4 instances; 8 total because there are 2 cores in the default config used
  during this run).

## Warnings (all pre-existing / benign)

These warnings were present in the first successful run and do not indicate
design bugs.  They are listed here so future runs can be diffed against this
baseline.

| Location | Message | Reason |
|----------|---------|--------|
| `controller.sv:75,85` | Truncated 32-bit value to 3 bits / 1 bit | Counter index assigned from integer literal; no data loss at runtime |
| `dispatch.sv:31,49,73,77,87` | Truncated 32-bit value to 3/8 bits | Same pattern — parameterized counter arithmetic |
| `registers.sv:68,69` | Truncated 32-bit value to 8 bits | `localparam` arithmetic on 8-bit register file |
| `kernel_memories.sv:294` | `_unused` assigned but never read | Intentional sentinel signal in auto-generated file |
| `SW[0..3]`, `SW[8]`, `KEY[1]`, `KEY[2]` | Input pin drives no logic | These switch/key inputs are deliberately unused in the current `de1_soc.sv` |
| Connectivity (1 hierarchy) | See `de1_soc.map.smsg` | Tied to the unused SW/KEY pins above |

## Artifacts generated (gitignored)

After `quartus_map`, Quartus creates:

```
quartus/db/               # compiled partition database (~100 files, ~3 MB)
quartus/incremental_db/   # incremental compilation cache
quartus/de1_soc.map.rpt   # full A&S report (~300 KB)
quartus/de1_soc.map.smsg  # suppressed messages
quartus/de1_soc.map.summary
quartus/de1_soc.flow.rpt
```

All of these are listed in `quartus/.gitignore`.  They are local build state
and should not be committed.
